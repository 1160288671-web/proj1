## 项目初始化（2026-07-18）

- **框架选型**：LangGraph，因为工作流是确定性的 DAG 流水线，不需要动态 task delegation
- **LLM 抽象**：不绑定具体提供商，通过 `src/llm/provider.py` 统一接口
- **目录结构**：采用 LOOPKIT VAULT 结构（.claude/ 规则层 + src/ 执行层）
- **实施顺序**：先搭骨架（Step 1-4），再逐个填充 Agent（Step 5-6）

## A1 子任务1：数据源配置体系（2026-07-19）

- **配置分离**：md 人读 / JSON 机读。A1 改读 `.claude/skills/a1-hotspot/datasources.json`，废弃解析 Markdown 表格
- **五平台实测**：微博、百度、头条、B站（热搜+热门）5 源直连可用；小红书无公开接口，第三方免费聚合均失效，`enabled=false` 待决策（自建/付费/放弃）
- **接口要点**：百度必须用 `platform=pc` 变体才有 hotScore；B站热门无热搜指数，以 `stat.view` 播放量代热度，`tname` 分区是最可靠领域标签
- **热度归一化**：各源口径不一（百万级 vs 千万级 vs 播放量），A1 只记录口径元数据（`heat_meta`），归一化归 A2，待积累样本后定方案
- **落盘**：`hot_raw.json` 由 A1 自行写入 `data/`，含 meta 信封 + source_stats；跨源重复保留（重复是热度信号，去重归 A2）
- **RSS 通道**：五平台均为 JSON API，RSS 类型暂不支持，v2 再说
- **备用通道**：60s 聚合（微博/头条/百度）已配置但默认关闭
- **实测结果**：一轮采集 90 条（微博20/百度20/头条20/B站热搜10/B站热门20），零错误
