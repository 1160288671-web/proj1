# A3 搜索 Query 生成规则

> LLM 辅助生成搜索词，代码去重执行检索。

## 单一选题

- 从 `topics.json` 的 `keywords` 字段直接扩展
- 扩展方式：同义词 / 相关词 / 缩写 → 组成 3–5 个搜索词

示例：
```
keywords: ["演唱会", "黄牛", "门票"]
→ queries: ["演唱会 门票 价格", "演出票务 市场", "黄牛 溢价 退潮", "演唱会经济 2026"]
```

## 组合选题

- 基于 A2 的 `combo_hypotheses` 转写为搜索词
- 每条假设生成 2–3 个搜索词
- 取假设中的实体词 + 关系词合并

示例：
```
假设: "演唱会受众与潮玩消费者高度重叠，均为 18-30 岁悦己消费人群"
→ queries: ["演唱会 受众 画像", "潮玩 消费者 年龄", "悦己消费 年轻人"]
```

## 合并去重

- 所有 query 合并后去重
- 去重标准：编辑距离 < 3 视为重复
