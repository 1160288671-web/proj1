# LOOPKIT VAULT — Claude 项目结构

> 一个人干整个团队的活，核心在于你给 AI 装了什么样的"操作系统"。  
> 这个 `.claude/` 文件夹结构，将 Claude 从聊天工具升级为严格训练的精英团队执行引擎。

---

## 目录结构一览

```
.claude/
├── CLAUDE.md              # 主合约：定义 AI 的角色和行为准则
├── settings.json          # 权限与配置：限制 AI 能做什么 / 不能做什么
├── hooks/                 # 自动反射：每次对话后自我复盘
│   ├── pre-tool-use.sh    # 工具调用前拦截：Guard bad calls（防止错误调用）
│   ├── post-tool-use.sh   # 工具调用后处理：Format + Log（格式化与日志）
│   └── stop.sh            # 终止钩子：Clean exit（干净退出）
├── agents/
│   └── verifier.md        # 子代理：在全新上下文中挑错的"警察 AI"
├── skills/                # 肌肉记忆：33+ 个可复用技能，随时调用
│   ├── agent-llm/         # Claude 模式与最佳实践
│   ├── debug/             # 问题分类 + 复现排查
│   ├── security/          # 审计 + 威胁分析
│   ├── frontend/          # React + 可访问性 (A11y)
│   ├── testing/           # Jest + 浏览器测试
│   ├── refactor/          # 安全代码重构
│   ├── docs/              # README 文档 + API 文档
│   ├── data/              # Pandas + SQL 数据处理
│   └── git-ops/           # PR 管理 + 代码合并策略
├── .mcp.json              # 外部工具连接：MCP 工具注册清单
MEMORY.md                  # 跨会话记忆：每一步决策和理由的实时记录
run.sh                     # 循环运行器：主工作流入口
install.sh                 # 一键安装：curl 一行命令启动
README.md                  # 项目简介：整体说明与快速上手
```

> **概览**：33 skills · 9 tracks

## 各组件详解

### 1. CLAUDE.md — 主合约（Main Harness / Contract）
- **作用**：定义 AI 的核心角色、行为准则与操作系统规则
- **本质**：整个系统的"宪法"，决定 Claude 以什么身份、按什么标准工作

### 2. settings.json — 权限与配置（Permissions + Hooks）
- **作用**：限制 AI 的行动边界
- **内容**：哪些工具可以调用、哪些目录可以访问、资源使用上限等
- **本质**：系统的"安全护栏"

### 3. hooks/ — 自动反射（Reflexes）
每次对话后自动触发的自我复盘机制，包含三个钩子：

| 脚本 | 触发时机 | 作用 |
|------|----------|------|
| `pre-tool-use.sh` | 工具调用前 | **Guard bad calls** — 拦截错误或不安全的工具调用请求 |
| `post-tool-use.sh` | 工具调用后 | **Format + Log** — 格式化输出并记录执行日志 |
| `stop.sh` | 会话终止时 | **Clean exit** — 确保资源释放、状态保存的干净退出 |

### 4. agents/verifier.md — 验证子代理（Shift-Notes Cop）
- **作用**：在全新上下文中运行的"警察 AI"，专门负责挑错和验证
- **机制**：与主 AI 隔离运行，避免思维定势，确保质量把关
- **本质**：独立的质量审查员

### 5. skills/ — 技能库（Muscle Memories: 33 Total）
像肌肉记忆一样随时调用的 33+ 个可复用技能，按领域分类为 9 个赛道：

| 技能目录 | 覆盖领域 | 说明 |
|----------|----------|------|
| `agent-llm/` | Claude 模式 | 与 LLM 交互的最佳实践和模式 |
| `debug/` | 调试排障 | 问题分类（Triage）+ 复现（Repro） |
| `security/` | 安全审计 | 代码审计 + 威胁模型分析 |
| `frontend/` | 前端开发 | React + 无障碍访问（A11y） |
| `testing/` | 测试覆盖 | Jest + 浏览器自动化测试 |
| `refactor/` | 代码重构 | 安全重写（Safe Rewrites）策略 |
| `docs/` | 文档撰写 | README + API 文档生成与维护 |
| `data/` | 数据处理 | Pandas + SQL 数据操作 |
| `git-ops/` | Git 工作流 | PR 管理 + 合并策略（Cubases） |

### 6. .mcp.json — 外部工具连接（MCP Tools Registered）
- **作用**：注册和配置外部工具（Model Context Protocol）
- **本质**：让 AI 能真正"动手干活"，连接文件系统、浏览器、数据库等

### 7. MEMORY.md — 跨会话记忆（Cross-Session Shift Log）
- **作用**：记录每一步决策和理由的"活记录"
- **特点**：跨会话持久化，确保长期项目的连续性
- **本质**：AI 的"工作日志"，让上下文永不丢失

### 8. run.sh — 循环运行器（The Loop Runner）
- **作用**：主工作流入口，驱动整个系统的持续运行
- **本质**：AI 团队的"自动运转引擎"

### 9. install.sh — 一键安装（One-Line Curl Bootstrap）
- **作用**：curl 一行命令完成整个环境搭建
- **本质**：零配置启动方案

### 10. README.md — 项目简介（Project Brief）
- **作用**：整体说明、架构概览与快速上手指南

---

## 系统核心理念

> **你不再跟 AI 聊天了。**  
> 你写好这个文件夹，文件夹替你执行 AI。  
> 每一次对话，AI 自动加载这些规则、权限、记忆、技能——像一支经过严格训练的精英团队在替你干活。

### 三大支柱：

1. **合约驱动（Contract-Driven）** — `CLAUDE.md` 定义角色，`settings.json` 划定边界
2. **反射闭环（Reflexive Loop）** — `hooks/` 确保每次交互都有复盘和校验
3. **肌肉记忆（Muscle Memory）** — `skills/` 积累领域专业能力，随叫随到
