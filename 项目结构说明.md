## 各组件详解

### 1. run.py — 主入口

命令行启动器，支持以下参数：

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--from` | 起始阶段 (A1-A5) | A1 |
| `--to` | 结束阶段 (A2-A6) | A6 |
| `--dry-run` | 干跑模式（Mock 数据 + Mock LLM） | 否 |
| `--api-key` | OpenAI API Key | 环境变量 `OPENAI_API_KEY` |
| `--base-url` | API Base URL | `https://api.openai.com/v1` |
| `--model` | 模型名称 | `gpt-4o` |

示例：

```bash
python run.py                          # 完整运行 A1→A6
python run.py --dry-run                # 干跑测试
python run.py --from A3 --to A5        # 从素材采集到视频制作
python run.py --model deepseek-chat    # 指定模型
```

### 2. src/ — 核心源代码

#### 2.1 main.py — VideoWorkflow 协调器

高层封装类，负责：
- LLM Provider 初始化（外部传入 > 环境变量 > Mock）
- 构建 LangGraph DAG
- 执行工作流并输出摘要

#### 2.2 graph.py — DAG 定义

定义 Agent 执行图：`START → A1 → A2 → A3 → A4 → [条件分支] → A5 → A6 → END`

唯一条件分支：A4 产出脚本为空时跳过 A5/A6，直接结束。

#### 2.3 state.py — 共享状态

`WorkflowState` 贯穿所有 Agent，各阶段读写字段：

| 阶段 | 产出字段 | 说明 |
|------|----------|------|
| 配置 | `config` | 运行时配置 |
| A1 | `hot_raw` | 原始热点数据列表 |
| A2 | `topics` | 最多 3 个选题 |
| A3 | `materials` | `topic_id → [Material]` 映射 |
| A4 | `scripts` | 脚本列表 |
| A5 | `video_outputs` | 视频产出（累加） |
| A6 | `publish_results` | 发布结果（累加） |
| 贯穿 | `workflow_id`, `stage`, `errors`, `dry_run` | 元信息 |

#### 2.4 agents/ — Agent 层

**基类设计：**

- `ExecutionAgent` — 纯代码逻辑，不使用 LLM（A1、A3、A5、A6）
- `JudgementAgent` — 核心判断依赖 LLM（A2、A4）

**各 Agent 职责：**

| Agent | 类型 | 核心功能 |
|-------|------|----------|
| A1 热点采集 | 执行型 | 从 datasources.json 配置的数据源拉取热榜，输出 `hot_raw.json` |
| A2 选题裁决 | 判断型 | 领域过滤→敏感过滤→LLM 四维打分→选出 3 个选题 |
| A3 素材采集 | 执行型 | query 生成→RSS 抓取→清洗→去重→信息密度排序→Top N |
| A4 编剧 | 判断型 | LLM 读素材→定体验定位→写口播稿→拆分镜（7-10 个） |
| A5 音画制作 | 执行型 | TTS 配音→BGM 选配→AI 生图→ffmpeg 合成渲染 |
| A6 发布打包 | 执行型 | 复制成片→生成 publish.json→标记 pending_human |

### 3. schemas/ — 接口定义（Pydantic）

定义 Agent 间严格的 JSON Schema 接口，核心模型：

| 模块 | 核心模型 | 对应阶段 |
|------|----------|----------|
| `meta.py` | `Meta`（通用信封） | 全部 |
| `topics.py` | `Topic`, `Score`, `TopicsOutput` | A2 输出 |
| `materials.py` | `Material`, `InfoDensity`, `Dedup`, `MaterialsOutput` | A3 输出 |
| `script.py` | `StoryboardScene`, `Experience`, `ScriptOutput` | A4 输出 |
| `production.py` | `ProductionOutput`, `TtsParams`, `BgmParams`, `Asset`, `RenderParams` | A5 输出 |
| `publish.py` | `PublishOutput` | A6 输出 |
| `performance.py` | `PerformanceOutput`, `Metrics` | A7 预留 |

**贯穿规则：**
- 每个输出模型必须包含 `meta: Meta` 信封
- ID 贯穿：`topic_id → material_id → script_id → video_id`
- 归因透传：体验类型、情绪标签从 A2/A4 逐级透传

### 4. tools/ — 工具模块

| 工具 | 功能 | 调用方 |
|------|------|--------|
| `tts.py` | TTS 配音合成 | A5 |
| `bgm.py` | BGM 选配（按情绪+BPM） | A5 |
| `image_gen.py` | AI 背景图生成 | A5 |
| `render.py` | ffmpeg 视频合成渲染 | A5 |
| `info_density.py` | 信息密度打分（0-9） | A3 |
| `dedup.py` | 去重聚类 | A3 |

### 5. utils/ — 通用工具

| 工具 | 功能 |
|------|------|
| `id_gen.py` | 统一 ID 生成：`tp-*` / `mt-*` / `sc-*` / `vd-*` / `wf-*` |

### 6. .claude/ — AI 配置与技能

- **CLAUDE.md**：AI 的主合约，定义角色、工作流、Agent 职责
- **agents/**：各 Agent 的 Markdown 规格说明
- **skills/**：专业知识库，Agent 运行时通过 `_load_skill()` 读取
- **settings.json**：权限与行为配置
- **hooks/**：自动化钩子脚本

### 7. data/ — 运行时数据

| 目录/文件 | 说明 | 产生方 |
|-----------|------|--------|
| `hot_raw.json` | 原始热点数据（含 meta 信封） | A1 |
| `output/` | 渲染后的 `.mp4` 视频文件 | A5 |
| `publish/` | 待发布打包目录（视频 + publish.json） | A6 |
| `reports/` | 编剧报告（Markdown） | A4 |
| `data/a1_data/probe/` | 热榜 API 探测样本 | 独立运行（tools/probe_a1.py） |

---

## 核心理念

1. **LLM 做判断，代码做执行** — 只有 A2、A4 调用 LLM，其余为确定性代码
2. **JSON 交接** — Agent 间用严格 Schema 传递数据，不做自然语言协商
3. **ID 贯穿** — 每份交接数据携带上游 ID，确保全链路可追溯
4. **归因透传** — 体验类型、情绪标签从 A2/A4 写入，逐级透传至数据回收
