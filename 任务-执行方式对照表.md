# 视频生成工作流 · 任务-执行方式对照表

> 依据：《视频生成工作流设计初稿.md》v0.1（开环版）
> 用途：划分 agent（LLM）与脚本的实现边界，作为开发任务拆解依据
> 标注约定：**LLM** = 必须调用大模型；脚本 = 纯代码/规则执行；工具 = 调用外部服务（TTS/生图/ffmpeg 等）；人工 = 人工介入

---

## A1 热点采集（纯脚本，0 次 LLM 调用）

| 任务 | 执行方式 | 说明 |
|---|---|---|
| 读取数据源配置 | 脚本 | 数据源清单做成配置文件 |
| 调用热榜 API / RSS 拉取条目 | 脚本 | HTTP 请求，设计稿明确"禁止 agent 临场写爬虫" |
| 原样结构化输出 `hot_raw.json` | 脚本 | 不做任何判断，只做格式封装 |

---

## A2 选题裁决（LLM 为主，脚本辅助）

| 任务 | 执行方式 | 说明 |
|---|---|---|
| 领域过滤（仅泛娱乐） | 脚本（可用） | 白名单是关键词表，可纯脚本匹配；边界模糊条目才需 LLM 兜底 |
| 敏感词过滤 | 脚本（可用） | 敏感词表匹配，规则确定 |
| 单一/组合候选生成 | **LLM** | 两两组合的穷举可用脚本，但"值得组合"的判断需 LLM |
| 关联假设生成（1–3 条/组合） | **LLM** | 第 7 节要求 LLM 生成"A 的 X 与 B 的 Y 因为 Z"句式假设，写不出即淘汰 |
| 四维打分（热度/安全性/可制作性/素材丰富度预判） | **LLM** | "预判"类打分无确定规则，须 LLM 按 rubric 评 |
| 加权求和、排序、放行 3 个 | 脚本 | 纯算术，权重可配置 |
| 输出 `topics.json` | 脚本 | 格式封装 + schema 校验 |

---

## A3 素材采集清洗（脚本为主，query 生成需 LLM）

| 任务 | 执行方式 | 说明 |
|---|---|---|
| query 生成（单一选题关键词扩展） | 脚本（可用） | 模板化扩展即可 |
| query 生成（组合选题：假设 → 3–5 搜索词） | **LLM** | 第 7 节要求从自然语言假设中抽实体词 + 关系词 |
| 素材抓取（RSS / 开放新闻源） | 脚本 | HTTP 请求 |
| 字数过滤（< 500 丢弃） | 脚本 | 第 6 节明确"全部由代码执行" |
| embedding 相似度判重（≥ 0.85 成簇） | 脚本 | 调 embedding 模型 + 阈值比较，非 LLM 判断 |
| 信息密度打分（满分 9） | 脚本（⚠️ 有隐患） | 设计稿声明代码执行，但见下方"待修正点 1" |
| 簇内择优、Top 8 排序、不足标记 `material_warning` | 脚本 | 纯排序逻辑 |
| 假设验证回填（`verified: false`） | 脚本 | 检索结果为空即标记 |

---

## A4 编剧（核心 LLM 环节，全链路 LLM 密度最高）

| 任务 | 执行方式 | 说明 |
|---|---|---|
| 阅读素材、理解事件 | **LLM** | |
| 定体验定位（主攻 1 + 辅攻 1，可推翻 A2 的 hint 但须写理由） | **LLM** | 对照第 5 节体验映射表判断 |
| 写口播稿（钩子 + 正文，棒读语感） | **LLM** | 核心创作任务，口语化短句 |
| 拆分镜（每镜选情绪标签、transition、bpm_hint） | **LLM** | 受控词表约束下的结构化生成 |
| 时长字数换算（目标时长 × 4.5 字/秒） | 脚本 | 纯算术，可在 prompt 前预计算注入 |
| 素材不足时收窄选题或放弃的决定 | **LLM** | 开环版允许 A4 跳过选题并在报告中说明 |
| schema 字段校验、Σduration 校验 | 脚本 | 第 3.0 节信封校验点 |
| 生成 `脚本报告.md`（人读，含体验定位理由） | **LLM** | 说明文写作 |

---

## A5 音画制作（固定管线，0 次 LLM 调用）

| 子任务 | 执行方式 | 说明 |
|---|---|---|
| TTS 配音 | 工具 | 调 TTS 服务，语速默认 1.0 |
| 音色选择（情绪标签 → 音色表） | 脚本 | 查受控映射表 |
| BGM 选配（标签检索 + BPM 对齐分镜节奏） | 脚本 | 曲库标签映射 + 数值匹配，明确"不做乐理分析" |
| 背景图生成 / 图库检索 | 工具 | AI 生图（生成模型，非 LLM 文本判断）或图库检索 |
| 全局风格后缀追加 | 脚本 | 字符串拼接，保证同条视频画风一致 |
| 字幕生成 | 脚本 | 从 TTS 时间轴自动切分 |
| 合成渲染 | 脚本 | ffmpeg 固定管线，"agent 只填参数不决策" |
| 记录 `production.json`（溯源用） | 脚本 | 参数落盘 |

> Agent 在 A5 的角色只是按序调度工具和填参数，理论上不需要 LLM 判断。

## A6 发布打包（脚本 + 人工，标题定稿可借助 LLM）

| 任务 | 执行方式 | 说明 |
|---|---|---|
| 标题 / 简介 / 标签定稿 | **LLM（轻量，可选）** | 若只做"从 A4 的 `title_options` 3 选 1"可用脚本；若需结合平台风格润色则需 LLM |
| 封面选择 | 脚本 / 人工 | 从分镜背景图中选取 |
| 文件打包进待发布目录 | 脚本 | 文件操作 |
| 生成 `publish.json` | 脚本 | |
| 人工放行 | 人工 | 设计稿明确"不自动上传" |

---

## A7 数据回收（本期只定义接口，混合模式）

| 任务 | 执行方式 | 说明 |
|---|---|---|
| CSV 导入、指标计算（完播率 / 5 秒留存 / 点赞率 / 评论率 / 分享率） | 脚本 | 纯统计 |
| 输出 `performance.json` | 脚本 | |
| 归因分析、写 `复盘报告.md` | **LLM** | 需把指标与体验 / 情绪标签关联解读 |
| 聚合归因标签、更新 A2 打分权重 | 脚本 | 数值更新，规则确定后可自动化 |

## 汇总

### 必须调 LLM 的任务（4 类，集中在 A2 / A4 / A7）

1. **A2**：组合关联假设生成 + 四维打分
2. **A3**：组合选题搜索词转写
3. **A4**：全部创作任务（体验定位、写稿、分镜、人读报告）
4. **A7**：归因复盘（本期不实现）

### 纯脚本 / 工具即可的任务

- **A1 全部**（配置读取、API 拉取、结构化封装）
- **A2 / A3** 的过滤、算术、检索、去重、排序
- **A5 全部**（TTS / BGM / 生图 / 字幕 / ffmpeg 均为工具调用 + 查表映射）
- **A6** 的打包与 `publish.json` 生成
- 各环节的 JSON 封装与 schema 校验

### 待修正的模糊地带（2 处）

1. **信息密度打分中"含因果解释（非纯情绪宣泄）：+2"** —— 数据、人名、时间线、引语可用正则 / NER 判定，但"是否含因果解释"本质是语义理解任务，纯代码很难可靠判定。建议：改为代理指标（如因果连接词出现频率），或显式承认该项需 LLM。
2. **A2 领域过滤、敏感过滤、A3 单一选题 query 扩展** —— 目前挂在 agent 名下，但完全可以脚本化。下沉后能显著降低 LLM 调用次数与输出不确定性。
